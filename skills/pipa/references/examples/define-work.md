# Define Work Examples

## Messy Client Thread

- `Working brief`: Client needs onboarding scope turned into a plan-ready brief.
- `In scope`: kickoff prep, acceptance checks, owner map
- `Out of scope`: implementation until scope is accepted
- `Acceptance checks`: every deliverable has owner, due date, and done check
- `Open questions`: budget owner and launch date are `TBD`
- `Decision needed`: approve scope baseline
- `Source`: pasted client thread

## Voice Request

Route to `pipa-huddle-beta` only if the user asks for live voice, walking session, huddle, or talking out loud.
