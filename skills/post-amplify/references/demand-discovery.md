# Answer Opportunity Discovery

Answer discovery finds public questions, threads, and recurring discussions that the source piece can help answer. The goal is useful participation, not link dropping.

## When To Use

Use answer discovery for practical/how-to, technical/build, operator, comparison, troubleshooting, and lesson-learned pieces with clear answerable topics.

For conceptual essays, announcements, or launches, run lighter discovery unless the user asks for answer opportunities and the query can remain public, relevant, and non-promotional.

## Search Strategy

Derive queries from the piece itself. Combine the core problem, technology, intended audience, and discussion venue.

Useful query patterns:

- `how to <task>`
- `<tool> vs <alternative>`
- `best way to <outcome>`
- `common problem with <technology>`
- `site:reddit.com <topic>`
- `site:news.ycombinator.com <topic>`
- `site:stackoverflow.com <error or topic>`
- `site:quora.com <topic>`
- `<topic> forum`
- `<topic> community`

Search for:

- live questions
- recent threads
- recurring questions
- forum posts
- subreddit matches
- newsletter or community fit
- Q&A pages where the source can add real context

## Community Rules

- Reddit, Hacker News, Stack Overflow, Quora, forums, Slack, Discord, and niche communities are manual-only by default.
- Do not post, comment, submit, vote, DM, create remote drafts, or interact with communities unless the user explicitly requests execution and the specific integration permits it.
- Include rules, norms, or self-promotion cautions when available.
- Avoid ranking by promotional opportunity alone; prefer relevance, recency, and usefulness.
- Suggest a contribution angle or answer outline, not a mass copy/paste comment.

## Query Privacy

Public source material can be used for public read-only search.

For pasted Markdown, local files, private/paywalled URLs, or unpublished drafts, ask before sending derived titles, excerpts, summaries, or keywords externally.

## Opportunity Output

For each opportunity, include:

- link or source
- community/source
- recency when available
- query that found it
- why it fits
- how the source helps
- suggested action: answer now, save, monitor, or skip
- rules/norms summary when available
- manual answer angle or outline

## Poor Fit Signals

Skip or mark low fit when:

- the thread is old, locked, off-topic, or promotional-sensitive
- the source only tangentially relates
- community rules prohibit self-promotion or require participation history
- answering well would require private or confidential details
- the action would feel like blind link dropping
