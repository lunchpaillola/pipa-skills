# Product

## Register

product

## Users

AI-assisted reviewers, delivery leads, builders, engineers, project managers, and operators who need to understand, steer, or continue work without staying fixed at a desk. They use Pipa skills while planning delivery, reviewing agent output, listening on a walk, or talking through a next step with an active agent.

## Product Purpose

Pipa skills turn messy delivery work into clear operator workflows for planning, updates, briefs, automations, handoffs, blockers, and voice sessions. Interfaces succeed when the next action is unmistakable, the user can trust the state of the workflow, and supporting context stays quiet until needed.

## Brand Personality

Focused, practical, calm, and low-brand. Pipa should feel like a working document or an expert operator beside the user, not a branded app shell, chatbot toy, dashboard, or decorative demo. For listening and voice surfaces, use the same visual language as `pipa-audio-brief`: restrained, transcript-aware, clear about state, and optimized for listening first.

## Anti-references

Avoid AI chatbot slop, SaaS dashboard clutter, developer tool chaos, repeated card grids, tab-heavy control panels, hidden session state, vague magic language, decorative gradients, badges, pills, metadata-heavy pages, and console-like noise that makes the next action unclear.

## Design Principles

1. Lead with the current job: listening, speaking, reviewing, or continuing work.
2. Make state unmistakable: ready, listening, sending, speaking, blocked.
3. Collapse supporting material: context, transcript, caveats, and handoff should be secondary.
4. Prefer reading and listening rhythm over component count.
5. Keep workflow surfaces calm enough for walking, reviewing, and decision-making.

## Accessibility & Inclusion

Use sensible accessibility defaults: readable type, strong contrast, keyboard-friendly controls, clear focus states, large tap targets on mobile, reduced-motion-safe behavior, visible status text for audio states, and text input fallback when speech recognition is unavailable.
